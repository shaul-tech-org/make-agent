# PRD: make-agent

## 1. 개요

### 제품명
make-agent

### 한 줄 설명
Claude Code의 `.claude` 설정을 웹 UI로 생성, 편집, 관리하는 도구.

### 배경
Claude Code는 `.claude/` 디렉토리 내의 설정 파일들(에이전트, 스킬, 규칙, 설정)을 통해 프로젝트별 행동을 커스터마이징한다. 현재 이 설정은 수동으로 마크다운과 JSON 파일을 작성해야 하며, 구조를 파악하기 어렵고, 파일 간 참조 관계가 복잡하다.

### 문제
1. **높은 진입 장벽**: .claude 디렉토리 구조와 파일 포맷을 알아야 설정 가능
2. **참조 관계 불투명**: 스킬→규칙→에이전트 간 의존성을 파악하기 어려움
3. **검증 부재**: 깨진 참조, 잘못된 frontmatter, 미사용 파일을 발견하기 어려움
4. **재사용 어려움**: 다른 프로젝트의 에이전트/스킬을 복사하려면 수동 작업 필요

### 목표
- CLI 없이 웹 브라우저에서 .claude 설정을 완전히 관리
- 시각적 관계 맵으로 설정 구조를 한눈에 파악
- 자동 검증으로 설정 오류 사전 방지
- 템플릿 기반으로 빠른 프로젝트 설정 생성

## 2. 대상 사용자

### Primary: Claude Code 사용자 (개발자)
- Claude Code를 프로젝트에서 활용하는 개발자
- .claude 설정을 처음 만들거나 기존 설정을 개선하려는 사용자
- 멀티 에이전트 구조를 설계하는 사용자

### Secondary: 팀 리드 / 아키텍트
- 팀 프로젝트의 Claude Code 설정을 표준화하려는 리더
- 에이전트 조직도와 스킬 파이프라인을 설계하는 사용자

## 3. MVP 범위

### 포함 (MVP)
1. **읽기**: 로컬 .claude 디렉토리를 파싱하여 웹 UI에 표시
2. **편집**: 에이전트, 스킬, 규칙, 설정을 UI에서 수정 → 파일에 저장
3. **생성**: 새 에이전트/스킬/규칙을 폼 기반으로 생성
4. **검증**: frontmatter 스키마, 참조 무결성, glob 패턴 검증
5. **시각화**: 에이전트 조직도, 스킬 파이프라인 다이어그램

### 제외 (Post-MVP)
- 템플릿 마켓플레이스 (에이전트/스킬 공유)
- GitHub 연동 (PR 자동 생성)
- 실시간 Claude Code 세션 모니터링
- 팀 협업 기능 (권한, 동시 편집)

## 4. 릴리스 전략

| 버전 | 범위 | 목표 |
|------|------|------|
| MVP (v0.1) | 로컬 .claude 읽기 + 뷰어 | "내 설정을 볼 수 있다" |
| v0.2 | 편집 + 생성 + 저장 | "UI에서 설정을 바꿀 수 있다" |

## 5. .claude 디렉토리 구조

```
.claude/
├── CLAUDE.md                    # 프로젝트 개요 (마크다운)
├── settings.json               # 권한, 플러그인 설정 (JSON)
├── agents/                     # 에이전트 정의
│   └── {name}.md              # YAML frontmatter + 마크다운
├── agent-memory/              # 에이전트별 메모리
│   └── {name}/MEMORY.md
├── skills/                    # 스킬 정의
│   └── {name}/
│       ├── SKILL.md           # YAML frontmatter + 마크다운
│       └── reference/         # 참조 문서
└── rules/                     # 규칙 정의
    └── {category}/
        └── {name}.md          # YAML frontmatter (paths) + 마크다운
```

### 파일 포맷 요약

| 파일 | 포맷 | 필수 frontmatter | 선택 frontmatter |
|------|------|-------------------|-------------------|
| CLAUDE.md | Markdown | 없음 | 없음 |
| settings.json | JSON | - | permissions, effortLevel, enabledPlugins |
| agents/*.md | Markdown + YAML FM | name, description, model | memory |
| skills/*/SKILL.md | Markdown + YAML FM | name, description | user-invocable, argument-hint |
| rules/**/*.md | Markdown + YAML FM | 없음 | paths (glob[]) |

## 6. 핵심 기능

### F1: 프로젝트 로딩
- 로컬 파일시스템 경로를 입력받아 .claude 디렉토리 파싱
- 모든 설정 파일을 읽고 구조화된 데이터로 변환
- 파싱 오류 시 경고 표시 (깨진 YAML 등)

### F2: 에이전트 관리
- 에이전트 목록 (이름, 모델, 역할, 메모리 스코프)
- 에이전트 생성/편집 폼 (frontmatter + 본문)
- 에이전트 조직도 시각화 (위임 관계)
- 에이전트 메모리 뷰어

### F3: 스킬 관리
- 스킬 목록 (이름, 설명, user-invocable 여부)
- 스킬 생성/편집 (frontmatter + 워크플로우 본문)
- 참조 문서 관리 (reference/ 디렉토리)
- 스킬 파이프라인 시각화 (의존성 그래프)

### F4: 규칙 관리
- 카테고리별 규칙 목록
- 규칙 생성/편집 (frontmatter paths + 본문)
- glob 패턴 테스터 (어떤 파일에 적용되는지 미리보기)
- 카테고리 추가/이동

### F5: 설정 관리
- settings.json 시각적 편집기
- 권한 매트릭스 (allow/ask 토글)
- 플러그인 토글
- CLAUDE.md 마크다운 에디터

### F6: 검증
- frontmatter 스키마 검증 (필수 필드, 타입)
- 참조 무결성 (스킬이 참조하는 규칙/문서 존재 여부)
- 미사용 파일 감지
- glob 패턴 유효성 검사

### F7: 시각화
- 에이전트 조직도 (트리 다이어그램)
- 스킬 파이프라인 (의존성 DAG)
- 규칙 적용 범위 매트릭스 (규칙 × 파일 패턴)

## 7. 기술 아키텍처

### 백엔드 (Python + FastAPI)
- **parser 모듈**: .claude 디렉토리 파싱 (YAML frontmatter, Markdown, JSON)
- **validator 모듈**: 스키마 검증, 참조 무결성 체크
- **writer 모듈**: 구조화된 데이터 → 파일 시스템 저장
- **project 모듈**: 프로젝트 로딩/관리

### 프론트엔드 (React + TypeScript)
- **대시보드**: 프로젝트 개요, 통계
- **에이전트 뷰**: 목록 + 편집 + 조직도
- **스킬 뷰**: 목록 + 편집 + 파이프라인
- **규칙 뷰**: 카테고리 트리 + 편집 + glob 테스터
- **설정 뷰**: JSON 에디터 + 권한 매트릭스

### 데이터 흐름
```
파일 시스템 (.claude/)
    ↓ 파싱
백엔드 (구조화된 데이터)
    ↓ REST API
프론트엔드 (UI 렌더링)
    ↓ 편집
백엔드 (검증 + 변환)
    ↓ 저장
파일 시스템 (.claude/)
```
