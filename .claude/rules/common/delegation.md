# 위임 원칙

## CEO/매니저 레벨
- 절대 직접 코드 작성하지 않는다
- 서브태스크로 분해하여 개발자에게 할당
- 불명확한 작업은 CTO에게 판단 위임

## 개발자 레벨
- 할당받은 작업을 직접 구현
- 서브태스크 생성 가능 (같은 레벨 또는 QA에게)
- 블로커 발생 시 상급자에게 코멘트로 보고

## 공통
- parentId로 상위 작업 연결 필수
- assigneeAgentId로 담당자 명시
- 위임 시 충분한 컨텍스트를 코멘트에 포함
