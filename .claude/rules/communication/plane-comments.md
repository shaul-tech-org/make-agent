# 에이전트 간 통신 — Plane 코멘트

## 원칙

에이전트 간 소통은 **Plane 이슈 코멘트**로 수행한다.
모든 행동, 결정, 질문, 결과를 코멘트에 기록하여 추적 가능하게 한다.

## 코멘트 형식

### 작업 시작
```html
<p><strong>🔄 작업 시작</strong><br/>
{작업 내용 요약}</p>
```

### 작업 완료
```html
<p><strong>✅ 작업 완료</strong><br/>
- 변경 사항 1<br/>
- 변경 사항 2<br/>
<br/>
커밋: {커밋 해시}</p>
```

### 위임
```html
<p><strong>➡️ 위임</strong><br/>
{담당 에이전트}에게 할당<br/>
이유: {왜 이 에이전트인지}<br/>
Sub-issue: {sub-issue 제목}</p>
```

### 블로커 보고
```html
<p><strong>🚫 BLOCKED</strong><br/>
원인: {블로커 원인}<br/>
필요한 도움: {무엇이 필요한지}<br/>
제안: {해결 방안}</p>
```

### 리뷰 요청
```html
<p><strong>🔍 리뷰 요청</strong><br/>
{CTO/QA}에게 리뷰 요청<br/>
변경 범위: {파일 목록}<br/>
테스트: {테스트 통과 여부}</p>
```

### 질문
```html
<p><strong>❓ 질문</strong><br/>
{상급자}에게: {구체적인 질문}</p>
```

## 규칙

- 코멘트 없는 상태 변경 금지
- 위임 시 반드시 코멘트로 컨텍스트 전달
- 블로커 즉시 보고 (방치 금지)
- 완료 시 결과 요약 필수
