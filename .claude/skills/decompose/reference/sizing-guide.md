# 작업 크기 판단 기준

## 적절한 작업 크기

하나의 작업(Task)은 다음 조건을 만족해야 한다:

| 기준 | 적절 | 너무 큼 | 너무 작음 |
|------|------|---------|----------|
| 에이전트 수 | 1명 | 2명 이상 | - |
| 완료 시간 | 1 Heartbeat 이내 | 여러 Heartbeat | 5분 미만 |
| 파일 수 | 1~5개 | 10개 이상 | - |
| 검증 가능 | 명확한 완료 기준 | "대충 되면" | - |

## 분해 수준

### Level 1: Epic (CEO 분해)
"사용자 관리 기능 만들어줘"
→ 기능별 분해 (인증, 프로필, 권한)

### Level 2: Story (CTO 분해)
"인증 기능 구현"
→ 기술별 분해 (DB, API, UI, Test)

### Level 3: Task (개발자 실행)
"회원가입 API 구현"
→ 직접 실행 가능한 단위

## 분해 원칙

**DO**: 한 에이전트가 한 번에 완료할 수 있는 크기로 분해
**DO**: 각 Task에 명확한 완료 기준(Done Definition) 포함
**DON'T**: 모호한 작업 ("잘 만들어줘")을 그대로 할당
**DON'T**: 너무 잘게 쪼개서 오버헤드 발생 (파일 1개 = Task 1개 수준)
