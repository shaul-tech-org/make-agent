# 의존성 분석 패턴

## 일반적 의존성 순서

```
리서치/조사 → 설계/기술 결정 → DB 스키마 → API → UI → 테스트
```

## 의존성 유형

### 순차 (Sequential)
반드시 선행 작업 완료 후 시작:
- DB 스키마 → API 엔드포인트 (테이블 없이 API 불가)
- API → UI (API 없이 UI 연동 불가)
- 구현 → 테스트 (코드 없이 테스트 불가)

### 병렬 (Parallel)
동시 실행 가능:
- 독립된 API 엔드포인트끼리 (회원가입 // 프로필)
- 독립된 UI 페이지끼리 (로그인 // 프로필)
- 다른 에이전트의 독립 작업

### 약한 의존 (Soft Dependency)
인터페이스 합의 후 병렬 가능:
- API 스펙 합의 → BE와 FE 동시 진행
- 테스트 케이스 설계 → 구현과 테스트 프레임 동시 진행

## 위험 패턴

**DON'T**: UI → API 역방향 의존 (API가 UI에 맞추는 구조)
**DON'T**: 순환 의존 (A → B → A)
**DON'T**: 3단계 이상 직렬 체인 (분해가 부족한 신호)
